# 🏛️ SaaS ESGAI 플랫폼 마스터플랜 6.0: DeepSeek 기반 Scale-Ready 설계

본 마스터플랜은 기존 로컬 AI 추론의 한계를 넘어, DeepSeek API를 통한 고가용성(High Availability) 확보와 상용 SaaS 수준의 보안/운영 정책을 통합한 최종 공정 지침서입니다.

---

## 1. 인프라 및 AI 추론 정책 (AI & Infrastructure Policy)

### 🛰️ 포트 및 라우팅 정책 (Fixed)
모든 외부 접속은 4600번 단일 게이트웨이로 수렴하며, AI 추론은 하이브리드 방식으로 운영됩니다.
| 서비스명 | 내부 포트 | 외부(Host) | 정책 |
| :--- | :---: | :---: | :--- |
| **Gateway (Caddy)** | 443 / 80 | **4600** | **단일 진입점.** `/api` → Backend, `/files` → Static FS 분기 |
| **SaaS Backend** | 8000 | (폐쇄) | **DeepSeek API 연동.** 비즈니스 로직 및 DB 핸들링 수행 |
| **Static Server** | - | (폐쇄) | 고용량 DOCX 직접 서빙. API 서버 리소스 보호 |
| **PostgreSQL 16** | 5432 | (폐쇄) | **외부 노출 완전 금지.** 전용 브릿지망 통신 |
| **Local AI Engine** | 11434 | (폐쇄) | 내부 전용. 보안 데이터 정밀 분석 및 백업 추론용 |

### 🧠 AI 추론 하이브리드 정책
- **1단계 (DeepSeek API)**: 동시 접속 대응 및 빠른 초안 생성을 위해 클라우드 기반 DeepSeek V3 API 사용.
- **2단계 (Local/API)**: 정밀 교정 및 데이터 정제 단계에서는 데이터 기밀성에 따라 로컬 Ollama 또는 DeepSeek API 선택 가동.
- **보안**: API 키는 시스템 환경 변수(`.env`)로 격리 관리하며, 로그 기록 시 개인정보는 마스킹 처리.
