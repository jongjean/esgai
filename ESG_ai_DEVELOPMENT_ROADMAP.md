# 🗺️ ESG ai 플랫폼 상세 개발 로드맵 (Development Roadmap)

본 문서는 **esgai** 전략적 마스터플랜을 바탕으로, 실제 시스템이 구축되고 서비스가 런칭되기까지의 모든 기술적/운영적 태스크를 주차별로 정밀하게 정의한 실행 가이드입니다.

---

## 📅 Phase 1: 인프라 기초 및 자립 환경 구축 (1주차)
> **목표**: 서비스 가동을 위한 독립적 런타임 환경과 데이터 저장소의 완벽한 격리.

- **Task 1.1: 디렉토리 트리 및 환경 파일 실체화**
  - `/home/ucon/esgai` 하위의 `web`, `engine`, `db` 폴더 구조 생성.
  - 서비스별 `Dockerfile` 및 `docker-compose.yml` 최종본 기재.
- **Task 1.2: 데이터베이스 스키마 초기화 (Port 5434)**
  - 네이티브 PostgreSQL 인스턴스 내 `esgai_db` 생성 및 전용 권한 부여.
  - `payments`, `reports`, `reviews` 테이블 스키마 물리적 생성.
- **Task 1.3: 인공지능 엔진(Ollama) 모델 적재**
  - `docker-compose up`을 통한 Ollama 컨테이너 가동.
  - `mistral` 또는 `llama3:8b-q4` 모델 Pull 및 추론 속도 벤치마킹.
- **Task 1.4: Caddy 서버 라우팅 설정**
  - `/esgai` 경로를 4600번 포트로 연결하는 리디렉션 정책 적용 및 SSL 확인.

**✅ Milestone 1**: `docker-compose ps` 상의 모든 서비스가 'Healthy' 상태로 전력이 안정 공급됨.

---

## 📅 Phase 2: 지능형 엔진 및 룰 엔진 실구현 (1~2주차)
> **목표**: ESG 평가의 신뢰도를 결정짓는 핵심 로직의 정합성 확보.

- **Task 2.1: 학회 기준 룰 엔진(rules.py) 개발**
  - 업종별(제조, 서비스, IT 등) 가점/감점 상숫값 매핑 테이블 구축.
  - 회사 규모(S/M/L)에 따른 평가 난이도 조정 로직 보완.
- **Task 2.2: AI 생성기(generator.py) 프롬프트 고도화**
  - 문장 생성 결과를 `E/S/G` 개별 JSON 필드로 강제 출력하는 로직 구현.
  - GPU 3GB 최적화를 위한 `num_predict` 파라미터 미세 조정.
- **Task 2.3: 엔진 내부 API 표준화**
  - `web` 서비스의 요청을 처리하는 `/generate`, `/customize` 핸들러 기능 완성.

**✅ Milestone 2**: 회사 정보 입력 시 AI가 항목별로 구조화된 정책 문장을 1분 내로 반환함.

---

## 📅 Phase 3: 비즈니스 서비스 및 보안 로직 완성 (2~3주차)
> **목표**: 결제와 기능 잠금을 연동하여 수익 창출이 가능한 시스템으로 격상.

- **Task 3.1: 단계별 서비스 전이(Level 10/30/60) 로직**
  - 현재 보고서의 상태 값을 DB와 연계하여 서비스 레벨을 동적으로 관리.
- **Task 3.2: 보안 결제 관리 시스템 (Payment Gateway)**
  - 서버 측 `payment_id` 검증을 통한 비인가 보고서 생성 시도 차단 정책 적용.
  - 결제 완료 시 해당 보고서의 `status`를 `paid`로 즉시 업데이트하는 후속 처리 기능.
- **Task 3.3: 관리자(Admin) 전용 백엔드 구축**
  - 결제 리스트 조회 및 검수 배정을 위한 내부 전용 API 완성.

**✅ Milestone 3**: 결제가 완료되지 않은 경우 PDF 다운로드가 보안 차단됨을 입증.

---

## 📅 Phase 4: 리포트 자동화 및 운영 UI 실체화 (3주차)
> **목표**: 사용자가 실제 돈을 지불할 만한 고품질 산출물 및 관리 환경 제공.

- **Task 4.1: WeasyPrint 기반 PDF 레이아웃 최적화**
  - `report.html`에 학회 표준 양식 반영 및 한국어 폰트 출력 이슈 완벽 해결.
  - 로고 삽입 및 CSS 스타일링을 통한 기업용 보고서 디자인 완성.
- **Task 4.2: 관리자 대시보드(admin.html) 기능 확장**
  - 결제 확인 버튼 및 검수 담당자 배정 드롭다운 메뉴 실체화.
  - 전역적인 시스템 상태(GPU 온도 등) 모니터링 기능 추가 제안.

**✅ Milestone 4**: 실제 기업 데이터를 넣었을 때 전문가 수준의 PDF 리포트가 10초 내 생성됨.

---

## 📅 Phase 5: 최적화, 검증 및 시장 런칭 (4주차)
> **목표**: 오류 없는 완벽한 상태로 상용 도메인 및 시장에 출시.

- **Task 5.1: 전체 시스템 통합 QA 및 부하 테스트**
  - 다중 접속 시 GPU 자원 경합 해결 상태 재점검 (세마포어 로직 확인).
- **Task 5.2: 운영 데이터 백업 및 모니터링 설정**
  - `server-bot` 연동을 통한 결제 발생 시 실시간 알람 설정.
- **Task 5.3: 마케팅 연동 및 정식 런칭**
  - 랜딩페이지(`/var/www/landing`) 앱 리스트에 **esgai** 카드 정식 등록 및 오픈.

**✅ Milestone 5**: 실제 유료 회원이 가입하여 첫 번째 결제가 발생하고 PDF가 다운로드됨.
