## 🚀 1. 제품 정의 및 비전 (Product Identity)

### 제품명: esgai (ESG Evaluation Service Engine)
> **"중소기업 및 공급망 대응 기업을 위한 ESG 자동 생성 및 실무 검토 SaaS 플랫폼"**

- **해결하고자 하는 문제**: ESG 공시 의무화로 인한 중소기업의 인력/비용 부담 및 전문 지식 부족.
- **핵심 가치**: 학회 기준의 신뢰도 높은 평가와 AI의 신속성을 결합하여 '가장 빠르고 정확한' ESG 보고서 제공.
- **주요 타겟**: 
  - 대기업 공급망 내 포함된 협력사 (중소/중견기업)
  - ESG 보고서 작성이 시급한 실무자
  - ESG 컨설팅 비용이 부담스러운 초기 창업 기업

## 🔄 3. 서비스 운영 시나리오 (Operation Workflow)

시스템과 사람이 유기적으로 협력하는 실제 서비스 흐름입니다.

1.  **유저 유입**: 회원 가입 후 기업 기본 정보 입력.
2.  **Level 10 자동 생성**: 룰 엔진 기반으로 즉시 초기 보고서와 정책 초안 제공.
3.  **결제 발생**: 사용자 판단 하에 **Level 30/60** 맞춤형 고도화 결제 (₩100,000).
4.  **검수 요청 자동화**: 결제 승인 시 시스템이 관리자 대시보드에 **'검수 준비'** 상태로 리포팅.
5.  **관리자 배정**: 운영 총괄자가 전문가(Reviewer)에게 해당 보고서를 수동/자동 배정.
6.  **학회 기준 보완**: 전문가 검수 후 상태를 `Complete`로 전환.
7.  **최종 PDF 다운로드**: 결제 및 검수가 완료된 데이터에 한해 PDF 출력 활성화.

## 🔌 4. 시스템 기술 명세 (Technical Specification)

### 🐋 인프라 자원 할당 (Conflict-Free Zone)
- **Web (Port 4600)**: 사용자 인터페이스, 결제 보안 로직, 관리자 콘트롤러.
- **Engine (Port 4610)**: 학회 기준 **Rule Engine**, AI Generator, PDF 렌더링.
- **Database (Port 5434)**: PostgreSQL 16 (인스턴스 분리 운영).

### 🧠 지능형 엔진 상세 (Revised Logic)
#### ① AI 생성기 구조 (generator.py)
기존의 통문장 반환 방식에서 **항목별 구조화 반환**으로 수정하여 PDF 데이터 정합성을 확보합니다.
```json
{
  "environment": "AI가 생성한 환경 분야 맞춤형 정책 문장...",
  "social": "AI가 생성한 사회 분야 실무 지침...",
  "governance": "AI가 생성한 투명 경영 정책 문장..."
}
```

#### ② 보안 결제 인증 (web/main.py)
단순 클라이언트 요청 플래그가 아닌, **서버 측 결제 ID 트래킹**을 통해 조작을 방지합니다.
```python
# 수정된 리포트 출력 보안 정책
if not db.verify_payment(payment_id, report_id):
    return {"error": "유효한 결제 정보가 없거나 검증되지 않았습니다."}
```
